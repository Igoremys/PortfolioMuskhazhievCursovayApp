# Code Smells

В проекте были обнаружены следующие проблемы:

1. **Дублирование логики**
   - Повторяющийся код трансформаций данных между DTO, Entity и Domain.
   - Повторное использование одних и тех же констант URL/ключей.

2. **Сильная связность между слоями**
   - Компоненты UI напрямую используют низкоуровневые DAO или API.
   - ViewModel выполняют слишком много логики, включая работу с сетью.

3. **Недостаточное разделение ответственности**
   - Репозиторий одновременно отвечает за бизнес-логику и детали синхронизации.
   - Контроллеры сервера содержат логику, которая могла бы быть в сервисном слое.

4. **Сложные условные конструкции**
   - Разветвления при обработке оффлайн/онлайн состояний в одном методе.
   - Плохая читаемость кода из-за вложенных `if` и `try/catch`.

5. **Недостаточное покрытие тестами**
   - Отсутствие модульных тестов для ключевых сценариев синхронизации и лайков.
   - Слабое тестирование переходов состояний ViewModel.
