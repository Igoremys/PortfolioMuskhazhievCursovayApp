# CI/CD

## Цели
- Автоматизировать сборку проекта.
- Проверять качество кода и покрытие.
- Запускать тесты на каждом коммите.

## Рекомендуемая схема
1. **Сборка проекта**
   - `./gradlew clean build`
2. **Запуск юнит-тестов**
   - `./gradlew :server:test`
   - `./gradlew :app:testDebugUnitTest`
3. **Проверка покрытия**
   - `./gradlew :server:jacocoTestReport`
   - `./gradlew :server:jacocoTestCoverageVerification`
4. **Статический анализ**
   - Проверка форматирования и стиля кода.
5. **Деплой**
   - Публикация артефактов на сервер или контейнер.

## Рекомендации для pipeline
- Выполнять проверки при каждом пуше в ветку `main`.
- Запускать интеграционные тесты после успешного прохождения юнит-тестов.
- Блокировать ветки, если `jacocoTestCoverageVerification` не пройден.
