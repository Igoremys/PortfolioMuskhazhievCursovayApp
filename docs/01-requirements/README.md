# Этап 1: Проектирование требований

## Общая информация
На данном этапе производится переход от бизнес-модели (Этап 0) к детальным системным требованиям. Акцент делается на функциональность мобильного приложения и серверной части в контексте паттерна PCMEF.

## Результаты этапа
1. ✅ Диаграмма Use Case (Системные прецеденты)
2. ✅ Domain Model (Концептуальная модель классов)
3. ✅ Спецификация прецедентов (Детализация ключевых Use Case)
4. ✅ Расширенный системный глоссарий (20+ терминов)
5. ✅ Матрица трассировки требований

## Ключевые особенности Траектории В
- Разделение требований на **Клиентские** (Android, Room, Jetpack Compose) и **Серверные** (Spring Boot, REST API).
- Обязательное наличие требований к **оффлайн-режиму** и синхронизации данных.
- Требования к безопасности (JWT, BCrypt).
