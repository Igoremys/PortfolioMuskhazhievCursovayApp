## UC3: Загрузка и публикация фотографии

| Поле | Описание |
|------|----------|
| **ID** | UC3 |
| **Название** | Загрузка и публикация фотографии |
| **Акторы** | Фотограф (основной), Мобильная ОС (вторичный) |
| **Описание** | Процесс выбора, сжатия и отправки фотографии на сервер с добавлением метаданных. |
| **Предусловия** | 1. Пользователь успешно аутентифицирован (валидный JWT).<br>2. Устройство имеет доступ к интернету (или фото будет сохранено в локальный кэш для отложенной синхронизации). |
| **Постусловия** | 1. Фотография сохранена в БД сервера.<br>2. Локальный кэш (Room) обновлен.<br>3. Фото отображается в ленте профиля автора. |
| **Основной сценарий** | 1. Фотограф нажимает кнопку "Добавить фото" (+).<br>2. Система запрашивает доступ к Галерее/Камере.<br>3. Пользователь выбирает изображение.<br>4. Система сжимает изображение на клиенте (до 1024px).<br>5. Пользователь вводит Название и Описание.<br>6. Пользователь нажимает "Опубликовать".<br>7. Система (Retrofit) отправляет POST-запрос на `/api/photos` с Multipart-данными.<br>8. Сервер сохраняет файл в хранилище, создает запись в БД.<br>9. Система получает ответ 201 Created и обновляет UI. |
| **Альтернативный сценарий 3а** | **Отсутствие сети (Оффлайн-режим):**<br>3а1. На шаге 7 система фиксирует ошибку сети (IOException).<br>3а2. Система сохраняет фото в локальную таблицу `pending_uploads` (Room DB) со статусом `PENDING`.<br>3а3. Система показывает уведомление: "Сохранено локально. Отправится при появлении сети".<br>3а4. WorkManager отслеживает появление сети и выполняет фоновую синхронизацию (UC6). |
| **Исключения** | 4а. Формат файла не поддерживается -> Показать ошибку "Только URL".<br>7а. Токен JWT истек -> Перенаправить на экран логина. |

## UC6: Оффлайн-кэширование и фоновая синхронизация

| Поле | Описание |
|------|----------|
| **ID** | UC6 |
| **Название** | Оффлайн-кэширование и синхронизация |
| **Акторы** | Фотограф, Мобильная ОС (BroadcastReceiver: Network State) |
| **Описание** | Обеспечение непрерывной работы приложения без доступа к сети и последующее слияние данных с сервером. |
| **Предусловия** | Приложение уже имеет закэшированные данные в Room Database. |
| **Постусловия** | Данные на устройстве полностью синхронизированы с сервером. Конфликты разрешены по стратегии "Server Wins". |
| **Основной сценарий** | 1. Приложение запускается или пользователь обновляет ленту.<br>2. Слой Foundation (Room DAO) мгновенно отдает последние данные в слой Presentation.<br>3. Слой Mediator (Repository) пытается сделать запрос к API.<br>4. При успехе: новые данные перезаписывают кэш (OnConflictStrategy.REPLACE).<br>5. При неудаче: исключение перехватывается, UI продолжает отображать кэш без ошибок. |

## UC1: Аутентификация и регистрация

| Поле | Описание |
|------|----------|
| **ID** | UC1 |
| **Название** | Аутентификация и регистрация |
| **Акторы** | Фотограф, Гость |
| **Описание** | Вход в приложение по email/паролю или регистрация нового аккаунта. |
| **Предусловия** | Устройство подключено к сети; email не используется (для регистрации). |
| **Постусловия** | Пользователь получает JWT и попадает на главный экран. |
| **Основной сценарий** | 1. Гость выбирает `Login` или `Register`.<br>2. Вводит email, пароль и дополнительные данные.<br>3. Система проверяет входные данные.<br>4. При успехе выдает JWT и переходит на главный экран.<br>5. При неудаче показывает ошибку. |
| **Исключения** | 1. Неправильный пароль → `401 Unauthorized`.<br>2. Email уже зарегистрирован → `400 Bad Request`.<br>3. Неверный формат данных → `400 Bad Request`. |

## UC2: Просмотр ленты фотографий

| Поле | Описание |
|------|----------|
| **ID** | UC2 |
| **Название** | Просмотр ленты фотографий |
| **Акторы** | Фотограф, Гость |
| **Описание** | Пользователь просматривает свежие фото из кэша и сети. |
| **Предусловия** | Приложение запущено, возможно есть локальный кэш. |
| **Постусловия** | Лента обновлена, UI отображает актуальные данные. |
| **Основной сценарий** | 1. Пользователь открывает главный экран.<br>2. Приложение показывает данные из Room.<br>3. Repository отправляет запрос к API.<br>4. При успешном ответе кэш обновляется и UI перерисовывается.<br>5. При отсутствии сети остаются старые данные и выводится статус оффлайн. |
| **Исключения** | 1. Сеть отсутствует → показ кэша.<br>2. Серверная ошибка → сообщение об ошибке, далее показ локальных данных. |

## UC4: Редактирование и удаление фотографий

| Поле | Описание |
|------|----------|
| **ID** | UC4 |
| **Название** | Редактирование и удаление фотографий |
| **Акторы** | Фотограф |
| **Описание** | Пользователь редактирует данные своих фото или удаляет их из портфолио. |
| **Предусловия** | Пользователь авторизован; фото принадлежит текущему пользователю. |
| **Постусловия** | Изменения сохранены на сервере и в локальном кэше. |
| **Основной сценарий** | 1. Пользователь открывает свой профиль или экран фото.<br>2. Нажимает `Edit` или `Delete`.<br>3. Система отправляет запрос на сервер.<br>4. При успехе кэш обновляется и UI показывает актуальный список. |
| **Исключения** | 1. Попытка изменить чужое фото → `403 Forbidden`.<br>2. Ошибка сервера → отмена операции и сообщение пользователю. |

## UC7: Поиск фото и пользователей

| Поле | Описание |
|------|----------|
| **ID** | UC7 |
| **Название** | Поиск фото и пользователей |
| **Акторы** | Фотограф, Гость |
| **Описание** | Пользователь ищет контент по имени, заголовку и описанию. |

| **Исключения** | 1. Никаких результатов → сообщение `Ничего не найдено`.<br>2. Ошибка сети → переключение на локальный поиск по кэшу. |

## UC8: Модерация контента

| Поле | Описание |
|------|----------|
| **ID** | UC8 |
| **Название** | Модерация контента |
| **Акторы** | Администратор |
| **Описание** | Администратор проверяет и принимает или отклоняет фото. |
| **Предусловия** | Администратор авторизован. |
| **Постусловия** | Отмеченные фото получают статус модерации и фильтруются в ленте. |
| **Основной сценарий** | 1. Администратор открывает панель модерации.<br>2. Выбирает фото для проверки.<br>3. Помечает как `approved` или `rejected`.<br>4. Приложение обновляет статус и ленту. |
| **Исключения** | 1. Ошибка сети → операция откладывается и показывается уведомление. |
