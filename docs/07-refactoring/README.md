# Рефакторинг проекта

В этом разделе описаны выявленные проблемы кода, примененные рефакторинги и результативные улучшения.

## Цели рефакторинга
- уменьшить связность модулей;
- улучшить читаемость и поддержку кода;
- убрать дублирование и сложные конструкции;
- сделать архитектуру ближе к PCMEF и MVVM.

## Содержание
- `code-smells.md` — основные «запахи» кода.
- `refactorings.md` — список выполненных рефакторингов.
