# Расширенный системный глоссарий

| № | Термин | Системное определение |
|---|--------|-----------------------|
| 1 | **JWT (JSON Web Token)** | Стандарт передачи данных, используемый системой для Stateless-аутентификации. Хранится в EncryptedSharedPreferences на клиенте. |
| 2 | **Multipart Request** | Тип HTTP-запроса, используемый для отправки бинарных файлов (фото) вместе с текстовыми полями. |
| 3 | **Room Database** | Локальная реляционная база данных на стороне Android-клиента, реализующая абстракцию над SQLite. |
| 4 | **DAO (Data Access Object)** | Паттерн, используемый в Room для описания SQL-запросов в виде интерфейсов Kotlin. |
| 5 | **Flow (Kotlin)** | Асинхронный поток данных, используемый для реактивного обновления UI при изменении данных в Room. |
| 6 | **Repository** | Архитектурный слой (Mediator), инкапсулирующий логику выбора источника данных (Сеть или Кэш). |
| 7 | **StateFlow** | Поток состояния в ViewModel, на который подписывается Compose UI для отрисовки экранов. |
| 8 | **Retrofit** | Библиотека-клиент для типобезопасного взаимодействия с REST API сервера. |
| 9 | **Interceptor** | Механизм OkHttp для автоматического внедрения JWT-токена в заголовки всех исходящих запросов. |
| 10 | **WorkManager** | Компонент Android Jetpack для выполнения отложенных фоновых задач (например, синхронизации фото). |
| 11 | **Coil** | Библиотека асинхронной загрузки и кэширования изображений в Jetpack Compose. |
| 12 | **OnConflictStrategy.REPLACE** | Стратегия Room для перезаписи существующих записей при получении обновленных данных с сервера. |
| 13 | **Stateless** | Архитектурный подход сервера, при котором состояние сессии не хранится на бэкенде (вся информация в JWT). |
| 14 | **BCrypt** | Алгоритм хеширования паролей, используемый на сервере (Spring Security) для безопасного хранения учетных данных. |
| 15 | **Pagination** | Механизм разделения выдачи API на страницы (Pageable) для экономии трафика и памяти устройства. |
| 16 | **Spring Data JPA** | Технология ORM на сервере для маппинга объектов Java на таблицы PostgreSQL. |
| 17 | **Entity (JPA/Room)** | Класс, аннотированный как сущность базы данных (`@Entity`), представляющий таблицу. |
| 18 | **DTO (Data Transfer Object)** | Объект, используемый исключительно для передачи данных между клиентом и сервером по сети. |
| 19 | **ViewModel** | Компонент Android Architecture, хранящий UI-состояние и переживающий повороты экрана. |
| 20 | **Jetpack Compose** | Современный декларативный тулкит для создания нативного UI на Android. |
| 21 | **Dependency Injection (DI)** | Паттерн внедрения зависимостей (Hilt/Koin), используемый для предоставления экземпляров Repository и DAO. |
| 22 | **Single Source of Truth (SSOT)** | Принцип, при котором UI получает данные только из локальной БД, а не напрямую из сети. |
