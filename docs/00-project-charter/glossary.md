# Бизнес-глоссарий предметной области

В таблице приведены ключевые термины, используемые в бизнес-модели и системных требованиях проекта.

| № | Термин | Определение | Синонимы / Английский аналог |
|---|--------|-------------|------------------------------|
| 1 | **Портфолио** | Совокупность лучших работ автора, структурированная для демонстрации профессиональных навыков. | Альбом, Галерея / Portfolio |
| 2 | **Медиа-контент** | Цифровые файлы (изображения, видео), загружаемые пользователем в систему. | Фото, Work, Asset |
| 3 | **Оффлайн-режим** | Способность приложения выполнять основные функции без активного подключения к сети Интернет. | Offline mode, Airplane mode |
| 4 | **Синхронизация** | Процесс выравнивания данных между локальным кэшем устройства и серверной базой данных. | Sync, Data reconciliation |
| 5 | **Лайк** | Единица социальной обратной связи, выражающая положительную оценку работы. | Like, Reaction, Favorite |
| 6 | **JWT-токен** | JSON Web Token — стандарт передачи данных для безопасной аутентификации между клиентом и сервером. | Token, Auth key |
| 7 | **Метаданные** | Сопутствующая информация о файле (дата съемки, локация, параметры камеры, теги). | EXIF, Meta info, Tags |
| 8 | **Пресет** | Набор заранее заданных настроек обработки или отображения фотографии. | Filter, Template |
| 9 | **Модерация** | Проверка публикуемого контента на соответствие правилам сообщества и законодательству. | Content review, Moderation |
| 10 | **Кэш** | Временное хранилище данных на локальном устройстве для ускорения доступа. | Local storage, Cache |
| 11 | **REST API** | Архитектурный стиль взаимодействия компонентов распределенного приложения через HTTP. | API, Web Service |
| 12 | **CRUD** | Аббревиатура базовых операций: Create, Read, Update, Delete. | Базовые операции |
| 13 | **Стейкхолдер** | Заинтересованная сторона, влияющая на проект или находящаяся под его влиянием. | Stakeholder |
| 14 | **Мобильный клиент** | Нативное приложение, установленное на мобильном устройстве конечного пользователя. | App, Mobile client |
| 15 | **Бэкенд** | Серверная часть системы, отвечающая за бизнес-логику и хранение данных. | Server side, Backend |
| 16 | **PCMEF** | Архитектурный паттерн разделения ответственности (Presentation-Control-Mediator-Entity-Foundation). | Архитектура слоёв |
