# Выполненные рефакторинги

## Основные изменения

- Вынесена бизнес-логика лайков и синхронизации из `ViewModel` в `PortfolioRepository`.
- Упрощены мапперы данных: DTO ↔ Entity ↔ Domain теперь реализованы единообразно.
- Связь между слоями приведена к схеме `Presentation -> Control -> Mediator -> Foundation`.
- Добавлен единый контракт `PortfolioRepository` для клиента, упрощающий тестирование.

## Улучшения структуры

- `PhotoViewModel` теперь только управляет состоянием UI и событиями.
- `Repository` отвечает за выбор источника данных и стратегию обновления.
- `PhotoDao` не содержит бизнес-логики, только операции CRUD.

## Результат

- Повышена читаемость кода.
- Уменьшено дублирование.
- Облегчено добавление новых сценариев (например, поиск и фильтрация).
- Созданы предпосылки для полного модульного тестирования ViewModel и Repository.
