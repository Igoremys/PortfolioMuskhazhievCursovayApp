# Этап 2: Архитектурное проектирование

## Общая информация
На данном этапе проектируется архитектура системы на основе паттерна **PCMEF** (Presentation-Control-Mediator-Entity-Foundation). 

## Адаптация PCMEF для Android Native (Траектория В)
Поскольку проект представляет собой клиент-серверное мобильное приложение, классический PCMEF адаптируется под современные реалии Android Jetpack:

| Слой PCMEF | Компонент Android | Технологии | Ответственность |
|------------|-------------------|------------|-----------------|
| **Presentation (P)** | UI Layer | Jetpack Compose | Отображение данных, обработка пользовательского ввода (нажатия, скролл). Не содержит бизнес-логики. |
| **Control (C)** | State Management | ViewModel + StateFlow | Хранение состояния UI, обработка событий от P, вызов методов Mediator. Переживает поворот экрана. |
| **Mediator (M)** | Domain Logic | Repository | "Единый источник истины" (Single Source of Truth). Решает, откуда брать данные (Сеть или Кэш). Инкапсулирует бизнес-правила. |
| **Entity (E)** | Domain Models | Kotlin Data Classes | Чистые бизнес-объекты (Photo, User), не зависящие от фреймворков (Room/Retrofit). |
| **Foundation (F)** | Data Layer | Room (DAO), Retrofit (API) | Инфраструктурный слой. Прямой доступ к локальной БД PostgeSQL и удаленному REST API. |

## Правило зависимостей
Зависимости направлены **строго сверху вниз**: 
`Presentation -> Control -> Mediator -> Entity <- Foundation`
Нарушение этого правила (например, обращение Foundation к Control) запрещено и приведет к штрафу.
