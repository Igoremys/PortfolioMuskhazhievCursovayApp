# Этап 3: Проектирование базы данных

## Общая информация
На данном этапе разрабатывается логическая и физическая модели данных для системы "Advanced Photo Portfolio App". Проектирование ведется с учетом требований клиент-серверной архитектуры (Траектория В).

## Требования к БД (согласно методичке)
1. **Нормализация:** Все таблицы приведены к 3-й нормальной форме (3НФ). Отсутствуют транзитивные зависимости и повторяющиеся группы.
2. **Целостность данных:** 
   - Четкое определение первичных (`PRIMARY KEY`) и внешних ключей (`FOREIGN KEY`).
   - Ограничения целостности: `NOT NULL`, `UNIQUE`, `CHECK`.
3. **Производительность:** Созданы индексы для полей, по которым часто выполняется поиск или JOIN (например, `author_id` в таблице фото, составной индекс для лайков).

## Результаты этапа
1.  ER-диаграмма (Логическая модель данных)
2.  DDL-скрипты (PostgreSQL для сервера)
3.  Описание стратегии ORM (Маппинг Entity → Таблицы для JPA и Room)
