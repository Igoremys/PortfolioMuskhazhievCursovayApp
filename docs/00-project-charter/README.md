# Этап 0: Инициация и бизнес-анализ

## 1. Паспорт проекта (Executive Summary)

### 1.1. Общая информация
- **Название проекта:** YPhoto
- **Траектория:** В (Мобильная разработка, Android Native)
- **Архитектура:** PCMEF (Presentation-Control-Mediator-Entity-Foundation)
- **Автор:** [Мусхажиев Игорь Александрович]
- **Дата начала:** 01.03.2026
- **Плановая дата завершения:** 30.05.2026

### 1.2. Бизнес-контекст
- **Проблема:** Фотографам и мобильным создателям контента не хватает единого мобильного приложения для быстрого управления портфолио, получения обратной связи (лайки, просмотры) и надежного хранения данных. Существующие решения либо требуют постоянного подключения к интернету, либо перегружены лишними социальными функциями.
- **Решение:** Разработка клиент-серверного мобильного приложения на Kotlin (Jetpack Compose) с бэкендом на Java (Spring Boot). Ключевая особенность — **оффлайн-режим** с использованием Room Database, что позволяет управлять портфолио без потери данных при отсутствии сети.

### 1.3. Цели проекта
1. Реализовать мобильное приложение с 5+ экранами и поддержкой CRUD-операций для фото.
2. Обеспечить работу приложения в оффлайн-режиме с последующей синхронизацией.
3. Разработать REST API (8+ эндпоинтов) с JWT-аутентификацией.
4. Спроектировать систему в соответствии с паттерном PCMEF.

### 1.4. Ключевые показатели успеха (KPI)
- **Производительность:** Загрузка ленты фото не более 2 секунд (при наличии сети). ⚠️ НЕ ПРОТЕСТИРОВАНО
- **Надежность:** Сохранность данных при обрыве связи (потеря 0 данных). ⚠️ УПРОЩЕННАЯ РЕАЛИЗАЦИЯ (нет очереди ожидания, оффлайн-загрузки отменяются)
- **Качество кода:** Покрытие серверной части тестами > 40%. ✅ ДОСТИГНУТО (сервис слой протестирован, интеграционные тесты отсутствуют)

### 1.5. Ключевые риски
- **Риск:** Задержки при загрузке изображений большого размера.
  *Стратегия:* Реализовать сжатие фото на клиенте и пагинацию на сервере.
- **Риск:** Конфликты данных при синхронизации после оффлайн-режима.
  *Стратегия:* Использование стратегии "Server Wins" или версионирование записей.
