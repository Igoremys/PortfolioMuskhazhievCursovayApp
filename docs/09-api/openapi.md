# OpenAPI

## Общая информация
OpenAPI используется для документирования REST API сервера. В проекте применяется SpringDoc OpenAPI, который автоматически генерирует спецификацию на основе контроллеров Spring Boot.

## Основные элементы спецификации
- `openapi` — версия спецификации.
- `info` — название проекта, версия, описание.
- `paths` — список эндпоинтов с методами и параметрами.
- `components` — модели данных, схемы ответов, параметры безопасности.

## Структура
- `PhotoDto` — схема ответа для фотографий.
- `UserDto` — схема ответа для профиля.
- `AuthRequest` / `AuthResponse` — схемы для авторизации.
- `ErrorResponse` — стандартный формат ошибок.

## Преимущества
- Позволяет автоматически проверять соответствие фактического API и документации.
- Облегчает создание клиентских SDK и тестовых запросов.
- Интегрируется со Swagger UI для наглядного просмотра.
